# Contributing to XRPL Impact Fund

Thank you for your interest in contributing to XRPL Impact Fund! We welcome contributions from developers, designers, and impact enthusiasts.

### Reporting Bugs

If you find a bug, please create an issue with:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Your environment (OS, Node version, etc.)

### Suggesting Features

We love new ideas! Create an issue with:
- Clear description of the feature
- Why it would be valuable
- Potential implementation approach
- Mockups if applicable

### Pull Requests

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes**
4. **Write tests** (if applicable)
5. **Follow code style** (ESLint + Prettier)
6. **Commit with clear messages**: `git commit -m "feat: add NGO verification API"`
7. **Push to your fork**: `git push origin feature/your-feature-name`
8. **Create a Pull Request**

### Commit Message Convention

We follow [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `style:` Code style changes (formatting, etc.)
- `refactor:` Code refactoring
- `test:` Adding tests
- `chore:` Maintenance tasks

Examples:
```
feat: add Impact NFT evolution system
fix: correct XP calculation for donations
docs: update architecture diagram
```
